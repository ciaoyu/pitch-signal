# 盘口异动智能解读引擎

## 设计思路

盘口变化是市场信息的聚合信号，但用户需要知道「为什么变」。
目标：盘口异动 → 自动关联可能原因 → 生成一句话解读。

---

## 一、异动检测规则

### 触发条件（任一满足即触发）：

| 维度 | 阈值 | 严重度 |
|------|------|--------|
| 主胜赔率变动 | ≥ ±8% in 30min | 🟡 关注 |
| 主胜赔率变动 | ≥ ±15% in 30min | 🟠 预警 |
| 主胜赔率变动 | ≥ ±25% in 30min | 🔴 重大 |
| 大小球线跳动 | ≥ ±0.5 球 | 🟠 预警 |
| 让球盘变动 | ≥ ±0.5 球 | 🟠 预警 |
| 成交量异常 | 偏离 5 分钟均值 3σ | 🟡 关注 |
| 博彩公司分歧 | Pinnacle vs 市场均值 >0.15 | 🟡 关注 |

---

## 二、原因推断逻辑

盘口变化 + 已知信息 → 推断原因

### 推断矩阵：

```
胜赔骤降（看好方更被看好）
├── 首发名单刚公布 → 对方核心缺阵 → "对方核心球员缺阵，市场信心向主队倾斜"
├── 天气突变 → 不利于对方风格 → "降雨增加，对方传控打法受限"
├── 场内消息 → 更衣室/训练消息 → "赛前消息利好主队"
├── 资金涌入 → 大额投注 → "检测到大资金入场，赔率被动调整"
└── 无明确原因 → "胜赔下降 +XX%，原因未明，建议关注赛前新闻"

胜赔骤升（看好方被看衰）
├── 首发公布核心缺阵 → "主队核心球员确认缺席"
├── 赛前热身受伤 → "赛前热身出现意外情况"
├── 对手首发超预期 → "对方排出强于预期的阵容"
└── 无明确原因 → "胜赔上升 +XX%，可能有利空消息待确认"

大小球线大幅上调
├── 双方防守核心缺阵 → "两队防线均有主力缺阵，进球预期上升"
├── 天气转好 → "降雨停止/转晴，利于进攻"
├── 战术泄露 → "预计双方将展开对攻"
└── 无明确原因 → "大小球线上调至 X.X，市场看好进球大战"

大小球线大幅下调
├── 核心射手缺阵 → "关键射手确认缺席，进球预期下降"
├── 暴雨/极端天气 → "恶劣天气将限制进攻效率"
├── 双方保守战术 → "预计谨慎开局，进球可能偏少"
└── 无明确原因 → "大小球线下调至 X.X，市场预期低进球"

角球盘异动
├── 边路球员伤缺 → "边路突破点缺失，角球预期降低"
├── 战术风格公布 → "对方排出密集防守，预计角球增多"
└── 无明确原因 → "角球盘变化至 X.X"
```

## 四、可用的数据触发源

当前系统已有的数据，可以实时喂给推断引擎：

| 数据点 | API | 可关联的盘口维度 |
|--------|-----|-----------------|
| 首发阵容公布 | ESPN lineup | 胜平负赔率 |
| 球员伤停 | ESPN injury | 胜平负 + 大小球 |
| 天气变化 | 天气 API | 大小球 + 角球 |
| 场馆草皮类型 | Venue DB | 让球盘 |
| 教练赛前发言 | 新闻 API | 全维度 |
| 大额投注 | Odds API volume | 胜平负 |

## 五、LLM Prompt 模板

当盘口异动触发时，用以下 prompt 生成解读：

```
你是一个专业的足球分析师。以下是世界杯比赛 "{home} vs {away}" 的盘口变化和目前已知信息。
请用一句话（不超过40字）解释盘口变化最可能的原因。

【盘口变化】
- 时间窗口：{timeWindow}
- 主胜赔率：{initialHome} → {currentHome}（{changePercent}）
- 大小球线：{initialOU} → {currentOU}
- 让球盘：{initialAH} → {currentAH}

【目前已知信息】
- 首发阵容变化：{lineupChanges}
- 球员伤停：{injuries}
- 天气情况：{weather}
- 近期状态：{recentForm}
- 历史交锋：{h2h}

【输出要求】
- 一句话解读（必填）
- 置信度：high/medium/low
- 对投注方向的建议：无
```
