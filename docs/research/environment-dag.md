## 1. 目标变量与建模位置

环境效应应进入**进球强度 λ**，而不是独立胜平负头（方法学笔记 §3.2）：

```
log λ_i = team_strength
        + β_alt   · (A_i − A_opp)      # 海拔：相对本队近期暴露差
        + β_heat  · (H_i − H_opp)      # 热湿 WBGT
        + β_turf  · (S_i − S_opp)      # 草坪（人工/天然）
        + β_travel· (T_i − T_opp)      # 旅行距离
        + β_tz    · (Z_i − Z_opp)      # 时区差
        + β_rest · (R_i − R_opp)      # 休息日差
        + β_et    · (ET_i − ET_opp)    # 加时分钟（上一场）
        + interactions
```

**对称约束（reviewer 硬性要求）**：每个变量以"本队近期暴露 − 对手近期暴露"的差进入，可同时影响本队进攻 λ 与对手防守压力；**不得只惩罚一边**。β 符号含义：正 β 表示该暴露对本队有利（如本队更适应高海拔），差值为负时自动转为不利。

## 2. DAG 节点

| 节点 | 含义 | 可得性（现状） |
|------|------|----------------|
| `Alt` | 场馆海拔（venue altitude, m） | 2026 场馆 100%；历史届 0% |
| `WBGT` | 湿球黑球温度（热湿综合） | 2026 部分；历史届 0% |
| `Turf` | 草坪类型（人工/天然） | 2026 场馆 100%；历史届 0% |
| `Travel` | 旅行距离 / 累计飞行 | 需 baseCamp+赛程推算；现状 0% |
| `TZ` | 时区偏移（基地→场馆） | 需 baseCamp 时区；现状 0% |
| `Rest` | 两场间隔休息日 | 可由赛程推算；现状 0%（未算） |
| `ET` | 上一场加时分钟 | 可由赛果+stage 推算；现状 0% |
| `Strength` | 球队实力先验（Elo） | 已有（elo-seed） |
| `λ_home`, `λ_away` | 双方进球强度（结果） | 模型输出 |

## 3. 因果 / 共线边（必须处理的混杂）

```
                 Strength ──┐
                    │       │ (球队越强→旅行越多，选择偏差)
                    ▼       ▼
   Alt ──────► λ_home    λ_away ◄──── Alt
    ▲            ▲  ▲              ▲  ▲
    │            │  │              │  │
   Travel ───────┘  │              │  └── Travel
    ▲               │              │
    │               │              │
   TZ ──────────────┘              └── TZ
    ▲                              ▲
    │   (长途旅行↔大时区差，强相关)  │
    └──────────────┬───────────────┘
                  Rest ◄── KO stage（淘汰赛休息更少）
                   ▲
                   │ (加时是比赛结果的函数 → 若以晋级为条件则成 collider)
                  ET ──► 但 ET 只在"上一场"定义，禁止用本场赛后回填本场赛前
   WBGT ──► λ     Turf ──► λ
     ▲                ▲
     │ (高温高湿↔大洲/草坪绑定)  │
     └──────── 相关 ──────────┘
```

关键共线 / 混杂（方法学笔记 §3.3）：

1. **Alt ↔ Travel**：高海拔场馆（如 Azteca 2200m）通常要求多数球队长途飞行 → `Alt` 与 `Travel` 正相关。单独解释 `β_alt` 会混入旅行效应。
2. **WBGT ↔ Continent/Turf**：高温高湿与特定大洲、草坪类型绑定；`β_heat` 可能吸收大洲效应。
3. **TZ ↔ Travel**：长途旅行伴随大时区偏移，`TZ` 与 `Travel` 共线。
4. **Rest ↔ KO stage**：淘汰赛轮次休息日更少；`Rest` 与赛制相关。
5. **ET collider**：加时分钟是上场比赛结果的函数；若以"是否晋级"为条件，则 `ET` 成为 collider，禁止在赛前预测中做此类条件化。
6. **Strength 选择偏差**：强队旅行更多，需在模型中控制 `Strength`（作为 λ 基线已含 Elo）。

## 4. 共线诊断与处理（验收要求）

- 估计前报告 **VIF** 与相关矩阵；报告后验相关（层级贝叶斯）而非单点估计。
- 对高度共线组（Alt/Travel/TZ、WBGT/Turf）使用**强正则化（Ridge / Elastic Net）或分层贝叶斯部分池化**，世界杯保持 held-out。
- **不以反直觉系数强行解释故事**：若 `β_alt` 符号与生理直觉相反，优先归因于共线/选择偏差，重新跑 sensitivity（去掉共线变量子集）后再下结论。
- 变量可用时间窗：1930–1960 年代几乎无法获得球员俱乐部分钟级暴露，早期比赛退化为基础模型，不得用国家平均海拔伪装成同等精度输入。

## 5. 第一阶段 / 第二阶段

- **第一阶段（本仓库当前可做）**：球队近期国家队比赛级暴露代理（近期比赛平均海拔、WBGT、人工草比例、旅行、时区、休息、加时分钟）。
