# PitchSignal 预测系统：投稿准备中的方法论整改报告

> 版本：2026-07-10  
> 用途：记录本轮讨论形成的研究判断、证据边界和未来论文必须披露的改进点。  
> 配套执行文档：[prediction-system-remediation-master-plan-2026-07.md](prediction-system-remediation-master-plan-2026-07.md)

## 摘要

PitchSignal 已具备真实赛程、赛果、Elo、Poisson、赔率快照、xG采集、阵容同步、实时事件、淘汰赛情报和赛后复盘等系统资产，但“有数据和代码”不等于“数据已经进入一个经过验证的概率模型”。当前生产概率仍混入多项手工规则：世界杯名义主客场加成、中文教练关键词、场馆容量、淘汰赛 λ 收缩、疲劳权重、红牌固定因子及两套启发式 confidence。与此同时，赔率、xG和lineup等真实数据处于采集、展示、关闭或接线不完整状态。

因此，未来论文不应把当前系统描述为一个已被964场历史回测整体证明有效的多信号模型。更准确、也更有发表价值的研究主张是：

> 建立一个可审计的足球概率预测系统，把“可改变概率的已验证信号”“前瞻shadow候选”“有来源的事实展示”“尚未验证的有趣假设”分层治理，并用历史回放和前瞻预测账本分别评价。

## 1. 两套真实证据不能混用

### 1.1 2026世界杯线上运行证据

当前生产校准接口的 `sampleSize=43`，其指标均来自本届世界杯已保存赛前预测与赛果：

- 方向准确率：27/43 = 62.79%；
- 三分类求和 Brier：0.5059；
- top-label ECE：0.1563。

所以“Brier 0.5059、ECE 15.6%是不是本届世界杯？”答案是：**是**。但它们评价的是截至当前的线上服务运行记录。由于快照表没有强制绑定不可变模型版本，这43场可能跨越多个代码/参数版本，不能自动写成“当前固定模型的OOS成绩”。

ECE 0.1563表示按当前top-confidence分箱后，预测置信与实际命中平均相差约15.6个百分点。当前高概率区间出现明显欠自信，40%–50%区间又有一定过度自信；43场足以展示现状，但不足以稳定训练复杂校准器，尤其不足以可靠评价平局校准。

### 1.2 历史回放证据

1930–2022的964场结果是：

- 558/964 = 57.88%方向准确率；
- Brier 0.5708；
- LogLoss 0.9644。

它是基于历史数据的walk-forward replay。它可以用于证伪公式、比较消融、发现年代和场景失效，但不是当时真实公开预测。早期世界杯更容易预测，并且当前回放还包含需要重新审计的主场、Coach、KO收缩等机制，因此57.88%不能再作为“完整引擎无缺陷”或“业界最准确”的结论。

论文必须将两套结果分表：`prospective operational ledger` 与 `retrospective historical replay`。

## 2. 数据有限时的产品与论文原则

能力有限、覆盖有限本身不是问题；隐藏覆盖、用中性伪概率补空、或把假设包装成模型才是问题。

系统应为每个模块标记四级证据：

| 等级 | 含义 | 是否可改概率 | 是否可展示 |
|---|---|---:|---:|
| Validated mover | 在严格时间外OOS中稳定改善proper score | 是 | 是 |
| Shadow candidate | 有真实数据和明确公式，尚在前瞻积累 | 否 | 可在实验层展示 |
| Descriptive fact | 有来源的赛程、伤停、环境、阵容、裁判等事实 | 否 | 是，必须标来源/覆盖 |
| Hypothesis | 有趣但未验证的战术或叙事判断 | 否 | 是，必须明确“假设” |

因此，用户希望看到的教练、环境、疲劳、裁判、球星状态、超级替补、熟悉度等内容不需要被删除。正确做法是：全部允许进入情报展示和Bot上下文，但只有通过验证的部分才进入概率。每个卡片应显示：

```text
source
asOf
coverage
evidenceLevel
usedInModel
missingReason
```

这会成为论文的系统治理贡献，而不是产品缺陷的遮羞说明。

## 3. 中立场与环境效应

### 3.1 中立场先纠错

世界杯数据中的home/away多数只是顺序字段。当前Elo `+100`、Poisson `×1.2`、容量Venue加成必须退出中立场；东道主只在球队国家等于场馆国家时成立。环境效应与主场优势是两类不同变量，不能混合。

### 3.2 环境效应的目标模型

环境应进入进球强度而不是独立胜平负头：

\[
\log \lambda_i(t)=
\text{team strength}
+\beta_{alt}A_i
+\beta_{heat}H_i
+\beta_{surface}S_i
+\beta_{travel}T_i
+\beta_{rest}R_i
+\text{interactions}
\]

其中变量必须描述“相对于本队近期暴露的差异”，而不是以国籍作粗糙代理。

### 3.3 本轮批注指出的三个落地难点

**覆盖年份。** 1930–1960年代几乎不可能获得主要球员俱乐部分钟级环境暴露。论文应明确变量可用时间窗；早期比赛退化为基础模型，不能用国家平均海拔伪装成同等精度输入。

**样本量。** 964场世界杯不足以稳定估计海拔、热湿、草坪、旅行、休息及交互。环境参数应从更大的国际比赛池或高水平俱乐部比赛池借力，世界杯保持held-out；使用强正则化、分层贝叶斯或部分池化。

**共线性与因果混杂。** 高海拔常伴随长途旅行和时差，高温高湿与大洲/草坪绑定。实施前必须画DAG，明确处理哪些混杂变量；报告VIF/相关结构/后验相关，不以反直觉系数强行解释故事。

### 3.4 分阶段工程方案

第一阶段使用可获得的球队级代理：近期国家队比赛平均海拔、WBGT、人工草比例、旅行、时区、休息、加时分钟。第二阶段才升级到球员近期俱乐部和国家队分钟暴露。所有变量对双方对称建模，可以同时影响本队进攻强度和对手防守压力。

环境超参数必须在训练窗内部验证，测试届不参与选择。最终即使整体增益很小，仍可作为有来源的解释模块展示；是否进入概率只由OOS结果决定。

## 4. Coach：立即纠错，未来重建

当前任期线性加分、中文风格词映射、缺数据仍输出confidence的Coach信号是伪特征，必须从概率中移除。这是纠错，不是普通改进。

真实教练研究需要：

1. 带有效起止时间的任职历史；
2. 纳入世界杯之外的预选赛、洲际杯、友谊赛以增加样本；
3. 控制球队实力、球员质量、对手、赛事、场地和换帅选择偏差；
4. 以相对预期进球/xG残差估计教练增值；
5. 对短任期做强烈分层收缩；
6. 战术风格来自阵型、压迫、推进和换人等行为，不来自形容词；
7. 在锁定未来比赛中评价LogLoss/Brier增量。

球员可用性、正式首发和近期状态的工程优先级高于教练。世界杯单个教练样本很少，即使完成研究，也可能只适合作为弱效应或展示项；这不是失败，而是实证结论。

## 5. 统一动态进球强度模型

当前实时系统有两条公式路径：Track A以比分/时间/红牌重新枚举Poisson，旧路径再以射门、射正、控球、牌和赔率做手工概率平移。前端又没有完整传红牌和淘汰赛状态。正确整改是：

1. 比分、时间、补时、红牌、淘汰赛状态作为状态骨架；
2. 用历史事件数据估计非均匀时间强度、比分状态、红牌时间/位置/交互；
3. xG、射门质量、换人、阵容进入未来进球hazard/λ，而不是直接“主胜加3%”；
4. 所有客户端、后台任务和monitor调用同一模型契约；
5. 旧手工公式退役，但原始事件数据保留；
6. 缺真实赛前基线时，不生成伪历史live概率。

示意：

\[
\log \lambda_i(t)=
f_{time}(t)
+\theta_{strength,i}
+\gamma_{score}(score,t)
+\gamma_{red}(red,t,position)
+\gamma_{xg}xG_{recent}
+\gamma_{lineup}lineup
+\gamma_{env}environment
\]

结构由足球过程提出，系数由历史事件数据估计。红牌固定`0.72`、加时固定1/3、点球默认技能边界等只能作为待检验基线，不能作为已证实事实。

## 6. xG、首发、赔率与Pressure

### xG

xG赛前用于近期攻防能力，实时用于已产生机会质量对未来λ的更新。当前collector健康不代表已接入主概率。免费路径是用开放射门事件训练自己的shot-xG；没有坐标时只能使用射正、禁区射门、绝佳机会代理，必须如实命名。

### 首发

首发未知不是`lineupUncertainty=0`。正式阵容公布前应对可能阵容做情景混合，公布后切换真实XI。每个球员状态必须有as-of时间，禁止使用赛后更新的身价、伤情或表现回填赛前。

### 赔率

赔率是独立市场概率与benchmark。当前每5分钟抓双区域、三市场导致额度快速耗尽，同时仅保留最后200条，是采集治理失败。未来按关键时点永久归档，先shadow，再由OOS决定是否进入constrained log-opinion pool。

### Pressure Index

Pressure可以作为有趣的过程展示，但当前35/20/20/15/10权重是手工描述指数，不是概率模型。研究问题应是：在控制比分、时间、红牌和赛前实力后，它能否改善未来5/10/15分钟进球概率？通过前保持`usedInModel:false`。

## 7. KO计划必须修订的全部问题

1. **KO-3**：轮次识别接线是正确的工程修复，但它同时激活了未经估计的R16/QF/SF/F λ收缩。应保留metadata，隔离shrinkMap，之后做预注册实验。
2. **KO-5**：休息日、旅行、加时是有价值事实；`weight=0.04`、组件权重和`maxTilt=0.025`是手工参数。先改为展示与shadow。
3. **KO-7/8**：点球历史和逐轮面板值得展示；球队技能、平滑先验、上下界必须从历史逐轮数据估计。90分钟赛果与晋级概率分开。
4. **DC**：`rho=-0.13`与`rho=0`只做了聚合比较，不能因CI重叠宣告结案。需按同一比赛做paired、按赛事cluster bootstrap；如果差异无实际意义，可保留在比分分布层或关闭，但结论必须来自正式检验。
5. **缺失信号**：缺席就是缺席，不再创建中性三元组；API和UI显示missing reason。
6. **情报价值**：停赛、裁判、超级替补、球星状态、熟悉度、比赛状态画像、经验、复盘教训即使不进概率，也应继续展示；必须标证据级别和数据覆盖。
7. **加时与疲劳**：历史回填后应将加时分钟作为疲劳的重要事实变量；旅行和休息不应因平均效应小而消失，但是否改概率必须实验决定。
8. **泄漏**：所有KO情报必须有as-of时间；赛后复盘教训不能倒灌到同场赛前预测。

## 8. 融合层的正式方向

对真正独立的概率源使用受约束的log-opinion pool：

\[
P(y)\propto e^{b_y}\prod_{s\in S_{available}}P_s(y)^{w_s},\quad w_s\ge0
\]

但环境、疲劳、教练、阵容等优先作为进球强度模型协变量，而不是每项各自产生一套H/D/A。Elo与使用Elo先验的Poisson高度相关，权重学习必须用Elastic Net或贝叶斯收缩；报告权重区间和跨届稳定性。缺失机制只有在OOS证明有信息时才能作为特征。

滚动协议建议：

```text
训练至2010 -> 测2014
训练至2014 -> 测2018
训练至2018 -> 测2022
历史模型冻结 -> 2026只作已锁定部分的前瞻/回放区分
```

由于2026结果已经看过一部分，不能再把整届2026描述成未见测试集。剩余比赛可以从新版本上线时开始形成真正prospective测试；下一届赛事继续积累。

## 9. 校准、不确定性与展示

校准需要每场开赛前H/D/A、模型版本、数据截点和最终90分钟结果。43场可以诚实展示，不足以隐藏不确定性或自动校准生产概率。

当前两套confidence应拆除或改名：

- 数据完整度：事实字段覆盖；
- 模型分歧：如JS divergence；
- 预测不确定性：由参数bootstrap/后验样本获得；
- 历史校准：类似场景的OOS可靠性。

真正区间应通过赛事级bootstrap重新拟合模型，获得一组组始终归一的概率三元组，再报告边际区间和经验覆盖。不能再对三个结果统一手工加减同一个半宽并称作95% CI。

网站推荐展示：

```text
2026公开预测账本：27/43，62.8%
Brier 0.5059 | ECE 15.6% | 模型版本覆盖说明
```

历史57.88%放在展开的“历史回放研究”中，并显示现代届次、阶段、东道主/中立场、实力区间和样本数。

## 10. 回填与前瞻账本

“补回填”分为三种，不能混淆：

1. **事实回填**：赛果、事件、红牌、阵容、加时、点球、天气，可以尽量补齐；
2. **外部概率回填**：历史赔率、FiveThirtyEight等其他模型，可作为benchmark；
3. **本模型回放**：用当时可得输入重跑，必须标`retrospective`；不能声称当时公开过。

真正丢失的赛前公开预测只有从Railway数据库、备份、日志、缓存、截图或发布记录中恢复；恢复不到就标`pre_missed`。从现在开始每条预测写不可变账本，并保留每次模型升级前后的shadow输出。

## 11. 投稿可以主张什么、不能主张什么

### 可以主张

- 一个防同日泄漏的世界杯历史回放协议；
- 一个真实2026前瞻运行账本，但需披露版本混合问题；
- 一套把validated、shadow、display-only、hypothesis分层治理的系统架构；
- 负结果、消融和数据覆盖限制的透明披露；
- 统一赛前/实时目标语义和可复现artifact。

### 当前不能主张

- 57.88%是业界最佳或证明完整系统可靠；
- Coach、Venue、Fatigue、Continental、Market Value已经历史验证；
- Brier 0.5059代表当前固定版本；
- 手工confidence是统计95%区间；
- xG、lineup、odds已经进入公开主概率；
- DC、淘汰赛收缩或红牌0.72已经科学结案；
- 2026整届仍是完全未见测试集。

## 13. 总结

本轮讨论形成的核心变化不是“给旧模型再加几个信号”，而是改变系统的证据纪律：

- 事实可以有限，但来源和覆盖必须透明；
- 有趣内容可以全部展示，但不能偷改概率；
- 参数可以复杂，但必须从时间正确的数据中估计；
- 历史回放、2026线上账本、retrospective重建、外部benchmark必须分开；
- 失败实验和小效应也要保留，因为它们本身是论文证据；
- 最终概率只来自统一、版本化、可复现、通过OOS检验的模型。

如果按这套路径完成，PitchSignal的投稿价值不只在某个准确率数字，而在于提供一个诚实、可审计、允许丰富情报存在但阻止未验证信号污染概率的足球预测系统范式。
