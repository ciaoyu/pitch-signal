### Endpoint Details

1. **`/health`**
   - Returns server health status with uptime and memory usage
   - No dependencies on external APIs

2. **`/api/schedule`**
   - Returns all matches from 2026-06-11 to 2026-07-03
   - Includes match status, scores, venue, and team information
   - Data source: ESPN API

3. **`/api/predict/:matchId`**
   - Returns prediction for a specific match
   - Requires valid matchId that exists in ESPN API
   - Returns error if match not found (ESPN 404)

4. **`/api/post-match-review/:matchId`**
   - Returns post-match review for a specific match
   - Requires valid matchId that exists in ESPN API
   - Returns error if match not found (ESPN 404)

5. **`/api/h2h/:matchId`**
   - Returns head-to-head data for teams in a specific match
   - Includes grouped matches (World Cup vs other competitions)
   - Includes summary statistics (win rates, streaks)
   - Includes recent matches and statistics

## Known Issues

1. **ESPN API 404 errors:** Some matchIds (e.g., 401770370) return 404 from ESPN API, causing prediction and review endpoints to return error responses with HTTP 200 status.

2. **Frontend HTML response:** The `/api/h2h/:teamA/:teamB` endpoint (with two team IDs) returns the frontend HTML page instead of JSON API response. This is expected behavior as the frontend handles this route.
