# 场馆与天气信息边界

## 当前产品规则

场馆模块只展示可核验事实：场馆、城市、世界杯比赛容量、海拔、屋顶、世界杯比赛草坪，以及天气 API 返回的温度、体感温度、湿度和风速。

以下旧功能已于 2026-07-15 撤销：

- 根据草坪类型给进攻、防守、控球或体能增加百分比；
- 宣称人工草坪“球速快、适合快速反击”；
- 根据草坪、海拔或天气给球队战术风格评级；
- 把未经校准的手工系数显示成量化“场地影响”。

这些关系可能值得研究，但当前仓库没有足够的比赛级样本、控制变量和样本外验证，不能作为比赛结论。

## 世界杯草坪口径

2026 世界杯比赛场馆使用以天然草为主体、注入合成纤维增强的比赛草坪。NFL/MLS 场馆平时使用的永久人工草皮，不代表世界杯比赛时的铺设方案。

`data/venues.json` 的 `grass` 字段记录世界杯赛事表面；场馆长期运营表面只能作为补充背景，不得覆盖赛事表面。

AT&T Stadium（世界杯名称 Dallas Stadium）当前记录：

- 世界杯草坪：混合增强天然草，临时铺设；
- 屋顶：可伸缩；
- 海拔：177 米；
- FIFA 公布的赛事容量：70,649。

## API 契约

`GET /api/venue/:id` 返回：

- `grass`：世界杯比赛草坪；
- `weather`：天气事实；
- `impact.quantified=false`：明确没有量化场地加成；
- `impact.details`：事实描述；
- `impact.disclaimerI18n`：中英文边界声明。

`GET /api/venue/:id/weather` 的 `styleFit.*.fit` 固定为 `unassessed`，直到有经过验证的场地—战术模型。

## 研究升级条件

若未来恢复场地影响模型，至少需要：

1. 比赛级实际草坪、屋顶状态和赛时天气；
2. 对手实力、球队能力、休息时间和场地中立性控制；
3. 明确的因果图与共线性审计；
4. 时间切分的样本外验证和不确定性区间；
5. 研究信号先进入 shadow ledger，证明增益后再进入公开页面。
